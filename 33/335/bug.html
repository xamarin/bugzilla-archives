Comment 3 Sebastien Pouliot 2011-08-20 11:22:40 UTC 
The error from the logs is misleading - since it's commonly associated with full-AOT limitations with generics which is not the case here.

While rebuilding the solution from the blog entry I got (took me a while to notice it):

Missing method MonoTouch.ObjCRuntime.Messaging::void_objc_msgSend_int_Boolean(intptr,intptr,int,bool) in assembly /Users/sebastienpouliot/Downloads/MTBarcodeExample/MTBarcodeExample/bin/iPhone/Debug/MTBarcodeExample.app/monotouch.dll, referenced in assembly /Users/sebastienpouliot/Downloads/MTBarcodeExample/MTBarcodeExample/bin/iPhone/Debug/MTBarcodeExample.app/LineaSDK.dll

That happened to me (and I guess it's the same for you) since I re-used the LineaSDK.dll from the .zip file. That .dll was generated with an old version of 'btouch'. I regenerated this library and
* the missing method (above) is gone while building the solution
* the System.ExecutionEngineException is gone from the device logs

The application still crash for me. However I don't have the libLineaSDK.a static library nor the hardware so that's likely the cause for my crash.

To regenerate the .dll simply
* download LineaSDK_4.cs form the blog article
* run "/Developer/MonoTouch/usr/bin/btouch ~/Downloads/LineaSDK_4.cs" from a terminal
* remove LineaSDK.dll from the project references
* add LineaSDK_4.dll to the project references

Please let me know if this solved your issue. I'm keeping this bug open until we fix the build process so the "missing method" is not so easy to miss.
