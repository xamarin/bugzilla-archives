Comment 55 Jon Douglas [MSFT] 2017-10-06 15:18:25 UTC 
For those experiencing this issue still, can you please do the following?

1. Clean your bin/obj folders of your project and remove any trace of the existing application from your device followed by the Mono Shared Runtime and Xamarin Android API Levels?

To do this, go to Settings -> Applications and remove these three applications:

- Your Application
- Mono Shared Runtime
- Xamarin.Android API-26 Support

This should put you in a fresh state to install items again. You can now try to rebuild and deploy.

2. Attach the IDE logs

You can grab these via the Help -> Zip Logs option in VS 2017. Specifically we would be after the "monodroid" logs as they will show devices being set and stopped.

3. If there's not much input in #2, Follow comment #20(https://bugzilla.xamarin.com/show_bug.cgi?id=56740#c20) and replace items with this diagnostic version which will put more output in the IDE logs:

https://www.dropbox.com/s/nytng838cj7synb/androidtools-bug59417.zip

Make sure to remove the mark of the web from these assemblies before installing.

Observations:

Based on your adb logs Tim, I am seeing socket connection refused items. Can you make sure that:

- The port is open on the destination
- There are no pending connections for the destination
- Something is not blocking access between the two?
