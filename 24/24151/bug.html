EXC_BAD_ACCESS in [WKWebView initWithFrame:configuration:] on 32-bit devices due to Unified API nfloat usage


This is very likely a duplicate of bug 23868.

I'm still filing a new bug anyway for 2 reasons:

(a) To record the precise stack trace of this particular error.

(b) To provide a test case that can be double-checked once the upcoming fix for bug 23868 is available.


Regression status: NOT a regression.


## Workaround

One known workaround for this problem is to build the app in Xamarin Studio on Mac.


## Steps to reproduce

Run the attached app on a 32-bit iOS 8 or iOS 8.1 device or simulator.

For example, the iPhone 4s iOS 8.1 simulator or the iPhone 5s iOS 8.1 simulator will show the problem.

Note: do *not* open the project properties before running the app. Opening the project properties can change the results due to bug 24150.


## Result

The app crashes while attempting to initialize the `WKWebView`.

## Expected result

The app should build and run without error on both 32-bit and 64-bit devices and simulators. If you build the app using Xamarin Studio on Mac, it behaves correctly.


## Additional information

The problem is caused by the `CGRect` input type that is passed into the `WKWebView()` constructor. If you manually invoke the corresponding `[WKWebView initWithFrame:configuration:]` selector, and pass in a `struct` that resembles `CGRect` but uses plain `float` fields instead of `nfloat` fields, then the crash does not occur.
