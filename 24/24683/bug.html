---Overview---
While attempting to test the sample app sent from this case:
https://xamarin.desk.com/agent/case/104741 (customer's issue appears unrelated.)

I noticed that several components were listed in the components section but were greyed out. The project was unable to build and I couldn't figure out how to load them. I tried several tests & experiments to try and determine the issue, but was unable to determine the cause / repo steps. 

Because of this, I made a detailed list of *ALL* the steps I was taking in the process:
1. Extract sample project
2. Open Xamarin Studio with project (XS was closed prior to this)
3. Both of the Android projects are unable to build. Fails due to ModernHttpClient not being resolved. 
4. Also, the following warnings are shown. (These are consistent with references missing from the References folder):
   a) GoogleAnalytics.Android - Reference not resolved
   b) HockeyApp.Android       - Reference not resolved
   c) ModernHttpClient        - Reference not resolved
   d) OkHttp                  - Reference not resolved
5. Components folder in both Android projects shows the following components greyed out
   a) Google Analytics
   b) HockeyApp for Android
   c) modernhttpclient
6. Looking at edit components:
   a) Google Analytics shows a description but appears not to be loaded that it’s not loaded.
   b) HockeyApp for Android is the same
   c) modernhttpclient only shows “Resolving component” message.
7. Both of the projects are able to add the already installed “Zing.Net.Mobile” component
8. Reboot Xamarin Studio
9. Changes:
   a) ZXing.Net.Mobile appears in red text
   b) Google Analytics appears to be loaded correctly
   c) HockeyApp for Android appears to have loaded in the library project but NOT in the other project.
10. Attempting clean of whole solution. No change.
11. Rebooting XS again.
12. Interestingly, HockeyApp for Android loaded all the way this time. 

After the above, all the components were loaded. Also upon deleting the components loaded version of the project; and doing a fresh extraction from the original project zip file, and reopening, these components loaded normally (as was expected originally.)

Comment 15 Matt Ward 2015-09-08 11:51:26 UTC 
I can repro this.

It seems to happen when a component is in multiple projects and the Components are not available in the machine's local cache (~/Library/Caches/Xamarin/Components).

When the "Already Downloaded, can't transition to Failed" error occurs then Xamarin Studio seems to be stuck. Closing and re-opening the solution does not allow the components to be downloaded to the solution's Components directory. Closing and re-opening Xamarin Studio can allow things to progress further but you may hit the same error again.

If the components are available in the machine's local cache then the problem does not seem to occur.
