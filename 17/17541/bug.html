Description Carlo Wahlstedt 2014-01-31 15:45:41 UTC 
When attempting to deploy an application from Xamarin Studio to an iPad the debugger does not always attach.


Steps to reproduce:
1. Make sure that you are using a multiple Desktop setup in OS X where Xamarin Studio is using it's own desktop.
2. Press Command + Enter to start debugging
3. Immediately press Control + Arrow key to switch to another desktop.
4. Stay on that desktop until you can tell, via the iPad, that the application has been deployed
5. Switch back to the desktop with Xamarin Studio


Result:
The application is running, but the debugger is not connected. The status window in Xamarin Studio says "Wait for debugger to connect (with nice animated icon). The Application Output window says:

Please ensure your device is connected...
Connected to: iPad
Launching /private/var/mobile/Applications/B3337048-8536-4E4D-8606-F21A4B272B3E/XamarinContainer.app -debugtrack -monodevelop-port 10000 -connection-mode usb


Expected Result:
The application should be running on the iPad with the debugger attached and outputting to the Application Output window.
