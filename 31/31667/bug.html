Description Cody Beyer (MSFT) 2015-07-06 12:44:09 UTC 
### Description

Using the latest versions (see below) of SDK tools and Xamarin.VisualStudio plugin, Visual Studio will become unresponsive when attempting to add item to the Resources/drawable folder.

### Steps to Reproduce

1. Ensure that the installed versions are in parity with those described below
2. Create a new Android app via: "Visual C# -> Android -> Blank App (Android)"
3. Right-click the "Resources\drawable\Icon.png" file and select "Exclude From Project"
4.  Right-click the "drawable" folder to attempt to add back the "Icon.png" file.

### Expected Results

The icon.png file will be removed and then re-added without any issue

### Actual Results

The UI will become inoperable on either steps 3 or 4

Comment 1 Cody Beyer (MSFT) 2015-07-07 11:46:28 UTC 
#### Possible Workaround

In further testing and communications, it appears that the hang only happens the first time that an image is added to a project. Once VS hangs, the task can be ended, and VS reopened, at which point adding images appears to work.

This has not been widely tested, so if you come across this bug report and are having this issue, please reply below with your results and version numbers

## Additional information to add to the bug report

If anyone can reproduce the behavior as described in comment 0 or comment 3, you can try collecting a call stack of Visual Studio's "Main Thread" from the hung instance of VS. I'd be particularly curious to see the call stack from an example where VS hangs the first time, but then succeeds after killing VS and retrying.


You can do this using a second instance of Visual Studio:

1. Start a second instance (a "new window") of Visual Studio.

2. Close any open solutions in the new instance of VS.

3. Select "Debug -> Attach to Process".

4. Select the original hung instance of `devenv.exe` from the list of "Available Processes".

5. Select "Debug -> Break All".

6. Make sure you have the "Debug Location" toolbar is enabled.

7. From the "Thread" drop-down menu in the "Debug Location" toolbar, select the "Main Thread".

8. Open "Debug -> Windows -> Call Stack", select all the lines, copy them, and paste them back into a text file attached on this bug report, or directly in a comment on this bug report.
