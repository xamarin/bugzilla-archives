Comment 29 Michael Carter 2015-06-17 10:01:31 UTC 
You can add me to the confirmed list.

As for being able to run CSG at the same time, the issue is that there are incompatible assemblies that both CSG and XS use. The ones in the GAC supersede all other assemblies, even ones in the same folder as the executable. This is one of the reasons the GAC is Evil(tm)

I found a workaround to be able to have them coexist. You basically have to remove the assemblies from the GAC. Apparently CSG actually does install the Mono.Cecil assemblies into the CodeSmith\Bin folder. So we just have to use gacutil to uninstall.

Unfortunately, nothing is ever easy when it comes to the GAC. Since Windows Installer is the one that installed the assemblies, gacutil won't let you uninstall, even with force enabled.

Turns out that Windows Installer keeps the assembly list in the registry. These are the values you need to remove. I've created a .REG file. I've added the corresponding file to re-add the values in case you want to rollback the change. See end of comment.

---%<--- snip here --->8--- 

Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Installer\Assemblies\Global]
"Mono.Cecil,version=\"0.9.5.0\",culture=\"neutral\",publicKeyToken=\"0738EB9F132ED756\",processorArchitecture=\"MSIL\""=-
"Mono.Cecil.Mdb,version=\"0.9.5.0\",culture=\"neutral\",publicKeyToken=\"0738EB9F132ED756\",processorArchitecture=\"MSIL\""=-
"Mono.Cecil.Pdb,version=\"0.9.5.0\",culture=\"neutral\",publicKeyToken=\"0738EB9F132ED756\",processorArchitecture=\"MSIL\""=-
"Mono.Cecil.Rocks,version=\"0.9.5.0\",culture=\"neutral\",publicKeyToken=\"0738EB9F132ED756\",processorArchitecture=\"MSIL\""=-

---%<--- snip here --->8--- 

You can then uninstall the assemblies with the following:

---%<--- snip here --->8--- 

set path=%path%;C:\Program Files (x86)\Microsoft SDKs\Windows\v8.1A\bin\NETFX 4.5.1 Tools
gacutil /u "Mono.Cecil,Version=0.9.5.0, Culture=neutral, PublicKeyToken=0738eb9f132ed756"
gacutil /u "Mono.Cecil.Mdb,Version=0.9.5.0, Culture=neutral, PublicKeyToken=0738eb9f132ed756"
gacutil /u "Mono.Cecil.Pdb,Version=0.9.5.0, Culture=neutral, PublicKeyToken=0738eb9f132ed756"
gacutil /u "Mono.Cecil.Rocks,Version=0.9.5.0, Culture=neutral, PublicKeyToken=0738eb9f132ed756"

---%<--- snip here --->8--- 

At this point, both CSG and XS will use their respective versions of the Mono.Cecil assemblies.

On a side-note, I believe it is really bad manners to have an assembly with the same name, version, and public key token yet be totally incompatible. And installing it in the GAC simply adds insult to injury.

It would be helpful for debugging in the future if perhaps in the Help > About > Show Details window, it would list all required assemblies and alert any that are in the GAC that are not the correct version. (Although you can't rely on version, you'd need to check file sizes, timestamps, or hashes).
