Comment 4 Rolf Bjarne Kvinge [MSFT] 2016-09-05 09:03:48 UTC 
Not using the IDE.

You can however invoke mlaunch manually:

1) Launch the app using Xamarin Studio.
2) Open a terminal and execute "ps aux | grep mlaunch". This will get you the complete mlaunch command.
3) Copy that command (you'll likely have to add quotes to account for spaces in the mlaunch path)
4) Add "--argument <your argument>" (without the quotes, and you can specify --argument multiple times) to the copied command and execute it.

Comment 27 Israel Soto [MSFT] 2017-03-10 09:25:27 UTC 
@Jeffrey,

I think there is a kind of regression with Arguments and Extra mlaunch arguments textfields.

If Arguments textfield has something and Extra mlaunch arguments is empty, Arguments is passed to mlaunch command but if it's on the contrary, Extra mlaunch arguments are not passed to mlaunch command. If both textfields has arguments, both are passed to mlaunch.

In other words, there must be something in Argument so Extra mlaunch arguments could be passed to mlaunch.
