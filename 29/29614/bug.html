Comment 15 Jeffrey Stedfast 2015-04-30 16:38:51 UTC 
It looks like whoever wrote the Publishing logic assumed that the Info.plist in the archive used the certificate's thumbprint for the "SigningIdentity" value as opposed to the common name (which is what is actually used), so when it fails to do a lookup of the certificate based on the "thumbprint", the identity menu is empty which causes the provisioning profile menu to be empty.

Based on that, if you edit the Info.plist at the root of your xcarchive, you should see something like this:

		<key>SigningIdentity</key>
		<string>iPhone Developer: Jeffrey Stedfast (YZ66U673MA)</string>

Change the value to be the SHA-1 fingerprint (without spaces) of your certificate and it should work.
Comment 16 Jeffrey Stedfast 2015-04-30 16:39:43 UTC 
I've fixed the code to do lookups based on the certificate's common name instead.
