To symbolicate the crash, you need the native debug symbols, the dSYM. It's a bundle (i.e. it's really a folder, but looks like a file) beside the app bundle in the build output directory of your project, so assuming you are using the Debug configuration, it would be "bin/iPhone/Debug/DinnerClub.app.dSYM".

The dSYM must be the one matching the exact build of the app bundle that crashed, so you may want to save the dSYM elsewhere to prevent newer builds overwriting them.

When you know where the dSYM and crash report are, go to Terminal.app and pass the crash report file and dSYM file to the symbolicatecrash tool, and it will decode the stack trace into actual function names and files:
symbolicatecrash -v path/to/log.crash path/to/app.DSYM
