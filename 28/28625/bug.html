Description Anubhav Ranjan 2015-04-01 07:59:49 UTC 
One of our customers is facing an issue when DataBinding is done for a Label where the Font attribute is bonded to Static Resource.

Description:
============

Our team had a problem with getting our Xamarin Forms app approved by Apple. They are telling us that the app crashes at startup and gives the symbolized stack dump at the bottom of this email.

After struggling for a while to find the problem, we finally found out that it only occurred on iPhone 6 (no other iPhones) and was triggered when data-binding the font of a label to a static resource as per below. Our workaround was to bind to the FontSize property instead. Just letting you know so you can fix this bug (at least we believe it is a bug) in future releases.

CODE Example:
============

XAML FILE


<Label
RelativeLayout.XConstraint="{ConstraintExpression Type=RelativeToParent, Property=Width, Factor=0.4}"
RelativeLayout.WidthConstraint="{ConstraintExpression Type=RelativeToParent, Property=Width, Factor=0.3}"
RelativeLayout.HeightConstraint="{ConstraintExpression Type=RelativeToParent, Property=Height, Factor=1}"
Text="{Binding ResultString}" Font="{StaticResource Font}" TextColor="#B6B6B4"/>


STATIC RESOURCE FILE

<?xml version="1.0" encoding="utf-8" ?>
<Application xmlns="http://xamarin.com/schemas/2014/forms"
xmlns:x="http://schemas.microsoft.com/winfx/2009/xaml"
x:Class="Livelox.Mobile.CoreUi.App">

<Application.Resources>
<ResourceDictionary>
<OnPlatform x:Key="Font" x:TypeArguments="Font">
<OnPlatform.Android>Medium</OnPlatform.Android>
<OnPlatform.iOS>Small</OnPlatform.iOS>
</OnPlatform>
</ResourceDictionary>
</Application.Resources>
</Application>
