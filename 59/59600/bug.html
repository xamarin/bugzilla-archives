Comment 2 Jon Douglas [MSFT] 2017-10-03 20:46:19 UTC 
I can CONFIRM this issue and I believe both https://github.com/Microsoft/msbuild/issues/1712 and https://github.com/Microsoft/msbuild/issues/608 are currently discussions for how to better relay this to developers so they can fix it.

So really I believe this happens because there is an explicit reference to Microsoft.CSharp 2.0.5.0 in the Xamarin.Android project when there is a reference to Microsoft.CSharp 4.3.0.0 in the netstandard library project from a library like JSON.NET.

Playing around with your sample, can you try to remove the explicit reference to the Microsoft.CSharp 2.0.5.0 item and then reinstall your Newtonsoft.Json/other library item so it will properly install the Microsoft.CSharp 4.3+ version? My question however is: Overriding this assembly that we distribute could end up breaking something else as the 2.0.5.0 version is from Mono and the 4.3+ version is from corefx. Given that mono is using corefx now, I think this should be okay. I am leaving this bug as CONFIRMED until someone else can take a gander at this.

Comment 8 dean.ellis 2018-02-26 10:38:39 UTC 
The bug #1261 [1] has been closed. Looks like MSBuild is correctly reporting this issue since it cannot decide if it should use Microsoft.CSharp  2.0.5/4.0.0. It should be down to the user to resolve this issue. If there is a dependency on 4.0.0 then that should be used since if we force 2.0.5 that would probably introduce runtime issues (as features will be missing). 

That said it seems the problem package (Newtonsoft.Json) was fixed.
