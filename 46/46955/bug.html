This normally happens if you have a Release version of the package and you try to deploy the debug version from the IDE. At that point the system can get into a state where the app does not show up in the Apps list on the device. Which makes it difficult to uninstall the app. In this case you have to uninstall manually from the add command prompt. You might even need to manually remove directories on the device. 

The latest version of Xamarin Android should handle this better. We attempt to detect configuration changes (Release->Debug) and then try too uninstall the previous app before installing the new one. 

The version you are running is a few releases old. I would recommend upgrading to Xamarin.Android 7.3.x and seeing if this is still a problem.
