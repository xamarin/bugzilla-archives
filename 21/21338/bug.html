Comment 1 Matt Ward 2014-07-15 12:08:02 UTC 
The problem is that a Xam.Mac project currently has a target framework of .NETFramework and not MonoMac so NuGet will not install anything from the MonoMac folder inside a NuGet package. Visual Studio will have the same behaviour as Xamarin Studio here.

As far as I am aware no Xamarin.Mac project nor any MonoMac project has used the MonoMac target framework even though NuGet supports it.

It might be better to get the ReactiveUI NuGet package changed so maybe it uses the an assembly in the Net45 folder that is compatible with the Mac.
Comment 2 Michael Teper 2014-07-15 12:13:05 UTC 
Matt, I don't think that's practical. The Net45 package is targeting Windows and WPF and has dependencies that don't exist on the Mac. How does NuGet handle fallback? In other words, if the project were to target Xamarin.Mac or MonoMac, would it fall back to Net45 if one of those folders were not found in a NuGet package?
Comment 3 Matt Ward 2014-07-15 12:45:42 UTC 
Agreed it is not going to work if ReactiveUI has WPF specific dependencies when targeting Net45 that are not supported on Mono.

The only fallback built into NuGet is that if a project targets .NET 4.5 then any version of .NET 4.5 or lower is considered compatible. So it could use Net45, Net40, Net20 assemblies in the NuGet package.

If the project had a target framework of MonoMac then only assemblies in a MonoMac folder could be used. If this was missing from the NuGet package then installation would fail.
