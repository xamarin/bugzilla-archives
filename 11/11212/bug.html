The `8badfood` code means that the iOS watchdog killed the application. 

That often (but not always) occurs when starting the application (the +/- 20 seconds you have to start an app and give back control to iOS). A common case is having a debugger breakpoint in that code.

That does not sounds like your case as LLVM does not support debugging. Maybe the code entered an infinite loop - which would also cause the watchdog to kill it at some point.
