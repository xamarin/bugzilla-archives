Comment 16 Alex Rønne Petersen 2013-11-22 05:59:54 UTC 
I have resumed work on armhf support lately. Single-precision and double-precision arguments work, and the runtime seems to run fine on the various CPU/FPU configurations I've tried.

We still need support for homogeneous floating point aggregates, but this only matters for P/Invoke when passing structs of pure float/double fields.

Branch is here: https://github.com/alexrp/mono/tree/armhf

Please post on this bug if you run into any issues with it.
